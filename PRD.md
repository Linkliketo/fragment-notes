# 碎片知识笔记应用 - 产品需求文档 (PRD)

---

## 1. 产品概述

### 1.1 产品定位

**产品名称**: 碎片笔记 (FragmentNotes)

**产品类型**: Web 应用（本地优先）

**核心价值**: 帮助用户整理和 管理碎片化知识，提供灵活的知识卡片组织方式，支持自由拖拽排序和结构化轴排序两种模式。

**目标用户**:
- 需要管理大量碎片化笔记的用户
- 喜欢视觉化知识组织的用户
- 追求简洁、低干扰界面的用户

## 2. UI/UX 设计规范

### 2.1 整体风格

| 特性 | 规范 |
|------|------|
| 色彩体系 | 黑白灰，低明度高灰度 |
| 主色调 | 纯黑 (#000000)、深灰 (#333333)、中灰 (#666666)、浅灰 (#999999)、极浅灰 (#E5E5E5)、纯白 (#FFFFFF) |
| 强调色 | 仅在必要时使用，使用低饱和度的灰蓝色 (#6B7B8C) 或灰紫色 (#7B6B7B) |
| 字体 | 无衬线字体，推荐使用 Inter、Noto Sans SC 或系统默认字体 |
| 字号 | 标题: 24px / 正文: 14px / 辅助文字: 12px |
| 圆角 | 卡片圆角 8px，按钮圆角 4px |
| 阴影 | 微弱阴影，仅在悬停时显示: `0 2px 8px rgba(0,0,0,0.08)` |

### 2.2 页面结构

```
┌─────────────────────────────────────────────────────┐
│  顶部导航栏 (固定高度 56px)                           │
│  [Logo] [库列表下拉菜单] [+新建库]       [设置]      │
├─────────────────────────────────────────────────────┤
│                                                     │
│                    主内容区域                        │
│                                                     │
│   (库内视图: 空白底面 + 文章卡片)                    │
│                                                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 2.3 组件设计

#### 2.3.1 顶部导航栏
- 高度: 56px
- 背景: #FFFFFF 或 #F5F5F5
- 底部边框: 1px solid #E5E5E5
- Logo 左侧，使用文字或简单图形
- 右侧: 设置图标按钮

#### 2.3.2 库列表
- 点击后展开下拉菜单
- 每个库名作为菜单项
- 当前选中的库高亮显示
- 支持右键菜单: 重命名、删除

#### 2.3.3 文章卡片

**基础样式**:
- 背景: #FFFFFF
- 边框: 1px solid #E5E5E5
- 内边距: 16px
- 最小宽度: 200px
- 最大宽度: 400px

**标题区域**:
- 始终显示在卡片顶部
- 字号: 16px
- 颜色: #333333
- 最多显示2行，超出省略

**内容区域**:
- 短篇（< 500字）: 显示全部内容
- 长篇（≥ 500字）: 显示前200字 + "展开全部" 按钮
- 行高: 1.6

**混乱排序模式**:
- 卡片位置: 自由拖拽
- 支持重叠: 后放置的卡片在上层
- 支持旋转: 可选，范围 -5° ~ 5°

**轴排序模式**:
- 卡片沿水平轴排列
- 间距: 24px
- 卡片宽度: 280px 固定

**卡片大小调整**:
- 默认尺寸: 宽280px × 高180px (4:3比例)
- 调整范围: 50% - 200%
- 档位设置: 50%, 75%, 100%, 125%, 150%, 175%, 200%
- 调整方式: 右键菜单 → 调整大小 → 选择档位
- 最小尺寸: 宽140px × 高90px
- 最大尺寸: 宽560px × 高360px

**卡片交互**:
- 单击(按压<200ms): 仅预览内容（不进入编辑）
- 长按(按压≥200ms): 拖动卡片移动位置（仅混乱模式）
- 双击: 进入编辑模式
- 右键: 打开右键菜单
- 拖拽: 移动卡片位置（仅混乱模式，需长按触发）

**右键菜单功能**:
- 编辑: 进入编辑模式
- 删除: 删除当前笔记
- 调整大小(二级菜单): 50%, 75%, 100%, 125%, 150%, 175%, 200%
- 重置大小: 恢复为默认100%
- 复制内容: 复制笔记内容到剪贴板
- 导出: 下载为Markdown文件

#### 2.3.4 排序切换控件
- 位置: 库视图右上角
- 形式: 两个切换按钮或开关
- 选项: "混乱排序" | "轴排序"
- 当前模式高亮显示

#### 2.3.5 轴线（轴排序模式）
- 颜色: #999999
- 粗细: 1px
- 样式: 虚线或实线
- 位置: 垂直居中

#### 2.3.6 新建/编辑文章弹窗
- 居中显示
- 背景遮罩: rgba(0,0,0,0.4)
- 弹窗宽度: 600px (最大)
- 包含: 标题输入框、Markdown编辑器、预览切换、保存/取消按钮

## 4. 用户流程

### 4.1 首次使用流程

```
1. 用户打开应用
   ↓
2. 显示空状态，提示"创建您的第一个库"
   ↓
3. 用户点击"创建库" → 输入库名 → 创建成功
   ↓
4. 自动进入新建的库，显示空白底面
   ↓
5. 提示"点击创建第一篇文章"
```

### 4.2 创建文章流程

```
1. 在库视图中，点击"+"按钮 或 点击空白区域
   ↓
2. 弹出编辑弹窗
   ↓
3. 输入标题 → 编写Markdown内容
   ↓
4. 点击"保存"
   ↓
5. 卡片出现在画布上（混乱模式在点击位置，轴模式在末尾）
```

### 4.3 拖拽排序流程（混乱模式）

```
1. 鼠标悬停在卡片上 → 显示可拖拽光标
2. 按住拖拽 → 卡片跟随鼠标
3. 释放鼠标 → 卡片放置在新位置
4. 位置自动保存
```

### 4.4 轴排序流程

```
1. 切换到"轴排序"模式
2. 显示水平轴线
3. 所有卡片沿轴排列
4. 拖拽卡片 → 在轴上移动位置
5. 释放后自动对齐到轴
```

## 5. 数据模型

### 5.1 库 (Library)

```typescript
interface Library {
  id: string;           // UUID
  name: string;         // 库名称
  createdAt: number;    // 创建时间戳
  updatedAt: number;    // 更新时间戳
}
```

### 5.2 文章 (Note)

```typescript
interface Note {
  id: string;           // UUID
  libraryId: string;    // 所属库ID
  title: string;        // 标题
  content: string;      // Markdown内容
  order: number;        // 排序位置（轴模式）
  positionX: number;    // 混乱模式X坐标
  positionY: number;    // 混乱模式Y坐标
  rotation: number;     // 旋转角度（混乱模式）
  zIndex: number;       // 层级（最后操作的卡片在最上层）
  sizeMultiplier: number; // 卡片大小倍数 (0.5-2.0)
  createdAt: number;    // 创建时间戳
  updatedAt: number;    // 更新时间戳
}
```

## 6. 技术方案

### 6.1 技术栈

| 层级 | 技术选择 |
|------|----------|
| 框架 | React 18 + TypeScript |
| 构建工具 | Vite |
| 状态管理 | Zustand |
| 存储 | IndexedDB (dexie.js) |

## 9. 术语表

| 术语 | 定义 |
|------|------|
| 库 (Library) | 笔记的集合单位，一个库包含多篇文章 |
| 文章 (Note) | 独立的笔记单位，包含标题和Markdown内容 |
